Assumption/Premise: Admin settings SHOULD NOT be split out into the Modules. Admin actions are role-based, and therefore, should be present or not present according to role.

 

Notes

  • Foundational Tenants will have super Admin capabilities and will not have any access to their tenants modules
  • If there is only one Link under an app, the app name IS the link (i.e. Batches and what is now currently Batches > Manage Batches)
  • If the list of Nav Items gets infinitely long, there might be reasons to ask why certain Apps have links at all. i.e. Should someone be forced to navigate to a Deposit via an Account? Someone can always use Global Search or use the Filter at the top of the drawer to find an app?
  • Nice to have: The arrow icons indicate opening the link in a new tab so that users can quickly have two screens in two tabs side by side. We kept to those to the right side to separate them purposefully.
  • Programming assumptions: The left nav bar would continue to be sticky to the viewport height and would NOT scroll with the rest of the page.
